# Contributing

Thank you for considering a contribution! Before you get started, please follow our contribution flow:

## 1. File an issue

Use the issue templates (bug, feature, docs) to describe what you'd like to work on. We can discuss scope, design, and any potential pitfalls.

## 2. Fork and branch

Fork the repo and `git clone`. Create a branch off `main` and use a descriptive branch name like `feature/add-xyz` or `fix/bug-xyz`.

## 3. Develop and test

Make sure all existing tests pass. Then add or update tests for your changes.

## 4. Submit a PR

Rebase onto the latest `main` and force-push to your branch to avoid merge commits. Then fill out the PR template, referencing the original issue. Make sure to describe your changes, any impacts on public APIs, and screenshots/logs if necessary.
